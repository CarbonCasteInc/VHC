# Reports Index

Files in `docs/reports/` are immutable or date-bounded evidence records. They
answer “what was observed in this evidence window?”, not “what is true now?”.

For current public-beta state, start with:

1. `docs/foundational/STATUS.md`;
2. `docs/ops/public-beta-operational-state.md`;
3. `docs/ops/public-beta-launch-readiness-closeout.md`.

When a report title says “current state,” interpret “current” at the date and
revision in that report. Never promote an old launch, recovery, bake, CI, host,
or mailbox result into present-tense evidence without a fresh readback.

Reports that are wholly superseded and likely to be mistaken for active control
documents belong under `docs/archive/`. Keep reports here only when their
bounded evidence remains useful to an active spec, runbook, or release gate.
